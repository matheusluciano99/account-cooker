## Ablations and failure cases

### Balance maintenance

Under the same seed and 120-second destination-local adversary:

| strategy | F1 | precision | recall |
|---|---:|---:|---:|
| one-to-one rotation | 0.25 | 1.00 | 0.14 |
| direct operator hub | 0.77 | 1.00 | 0.62 |

### Fee-payer funding

| policy | F1 | precision | largest cluster | anonymity set |
|---|---:|---:|---:|---:|
| omitted from ledger | 0.25 | 1.00 | 0.05 | 1.0 |
| operator hub | 1.00 | 1.00 | 0.20 | 1.0 |
| dedicated funder | 1.00 | 1.00 | 0.19 | 1.0 |
| 12 shared relayers | 0.76 | 0.61 | 0.29 | 1.5 |

The operator-hub and dedicated-funder cases are deliberate negative results: funding
re-links the fleet completely. The shared-relayer result has lower precision because the
heuristic over-merges different operators; it must not be read as a 0.76 privacy guarantee.

```json
  "estimated_network_fees_lamports": 62860000,

```

The 5,000-lamport value is an explicit example assumption, not a network quote.
Transferred value is gross cover/internal volume, not net spend. This ledger omits
fee-payer funding, so the estimate also omits those funding transactions and fees.

## Regression caught during verification

An initial chronology fix anchored each agent's next wake to the global observable clock.
The scale test rejected it because a 1,000-agent run fell to 207,667 records. The runtime
now keeps a per-agent logical schedule while monotonically clamping only emitted chain
timestamps. A fast `aggregate_activity_scales_with_the_fleet` regression test was added,
and the heavy proof returned to 2,250,843 records.
